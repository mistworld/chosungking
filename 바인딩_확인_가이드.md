# 바인딩 실제 동작 확인 방법

## ✅ 실제 바인딩이 작동하는지 확인

### 1. API 테스트
브라우저에서 직접 테스트:

```
https://chosung-game-v2.sucksuck1114.workers.dev/api/rooms
```

- **200 OK + JSON 데이터** 나오면 → KV 바인딩 정상 작동 ✅
- **404 에러** 나오면 → 바인딩 문제 또는 라우팅 문제

### 2. Worker 코드에서 확인
- worker.js에서 `env.ROOM_LIST`, `env.WORD_CACHE`, `env.GAME_STATE` 사용
- 이들이 없으면 코드 자체가 에러날 것

### 3. 배포 로그 확인
배포 시 "Your Worker has access to the following bindings:" 메시지가 나왔다면
→ 바인딩은 정상적으로 적용된 것

---

## 🔍 Dashboard "No connected bindings" 의미

Dashboard의 "Bindings" 탭은:
- **수동으로 추가한 바인딩**만 표시
- wrangler.toml로 배포한 바인딩은 여기에 안 보일 수 있음
- 실제로는 배포 로그에서 확인 가능

---

## 💡 실제 확인 방법

위 API URL을 브라우저에서 직접 접속해보세요!
- 정상 작동하면 → 바인딩 OK, 문제는 Assets만
- 404 나오면 → 바인딩도 문제
