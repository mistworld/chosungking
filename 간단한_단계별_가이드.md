# 간단한 단계별 가이드

## 현재 상황
- `chosung-game-do`가 목록에 없어서 다시 배포함
- 이제 `chosung-game-state-do`로 배포해야 함

## 해야 할 일 (단계별)

### 1단계: 배포 명령어 실행

CMD에서 다음 명령어 실행:

```cmd
wrangler deploy --config wrangler-do.toml
```

### 2단계: 배포 메시지 확인

다음 메시지가 나와야 합니다:
```
Total Upload: ...
Your Worker has access to the following bindings:
Binding                             Resource
env.GAME_STATE (GameStateRoom)      Durable Object
```

### 3단계: 배포된 Worker 확인

```cmd
wrangler list
```

이 명령어로 `chosung-game-state-do`가 목록에 있는지 확인하세요.

### 4단계: Dashboard 확인

1. Cloudflare Dashboard 열기
2. **Workers & Pages** → **Overview**
3. `chosung-game-state-do` Worker 찾기
4. 보이면 성공!

### 5단계: Pages에서 바인딩

1. **Pages** → 프로젝트 선택
2. **Settings** → **Functions** → **Durable Object Bindings**
3. **Add binding** 클릭
4. 입력:
   - **Variable name**: `GAME_STATE`
   - **Durable Object**: 드롭다운에서 `chosung-game-state-do` 선택
   - **Class name**: `GameStateRoom`
5. **Save**

## 중요 사항

- Worker 이름이 `chosung-game-state-do`로 변경되었습니다
- 이전 이름 `chosung-game-do`는 더 이상 사용하지 않습니다
- 배포할 때 `wrangler-do.toml` 파일을 사용하세요
